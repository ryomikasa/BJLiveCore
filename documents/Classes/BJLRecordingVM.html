– setRecordingAudio:recordingVideo:

开关音视频

- (nullable BJLError *)setRecordingAudio:(BOOL)recordingAudio recordingVideo:(BOOL)recordingVideo

Parameters

recordingAudio

YES：打开音频采集，NO：关闭音频采集
recordingVideo

YES：打开视频采集，NO：关闭视频采集

Return Value

BJLError: BJLErrorCode_invalidCalling 错误调用，如学生在非上课状态下开启音视频、在音频教室开启摄像头

Discussion

上课状态才能打开音视频，参考 roomVM.liveStarted

上层自行检查麦克风、摄像头开关权限

上层可通过 BJLSpeakingRequestVM 实现学生发言需要举手的逻辑

– remoteChangeRecordingWithUser:audioOn:videoOn:

老师: 远程开关学生音、视频

- (nullable BJLError *)remoteChangeRecordingWithUser:(BJLUser *)user audioOn:(BOOL)audioOn videoOn:(BOOL)videoOn

Parameters

user

对象用户，不能是老师
audioOn

YES：打开音频采集，NO：关闭音频采集
videoOn

YES：打开视频采集，NO：关闭视频采集

Return Value

BJLError: BJLErrorCode_invalidArguments 错误参数； BJLErrorCode_invalidUserRole 错误权限，要求老师或助教权限。

Discussion

打开音频、视频会导致对方发言状态开启

同时关闭音频、视频会导致对方发言状态终止
